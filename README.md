# מאגר שיעורי תורה

A full-stack Torah lecture library: a Python backend that fetches YouTube playlists and produces `categorized_videos.json`, and a React + Vite frontend deployable on Vercel.

## Backend Setup

### How it works

The backend has three files:

- **`playlist_utils.py`** — uses `yt-dlp` to scrape all playlists from the two Rav Butbul YouTube channels, then categorizes them by keyword matching against playlist titles.
- **`playlist_videos_utils.py`** — fetches video metadata via the YouTube Data API v3. It's **incremental**: it loads the existing `categorized_videos.json` cache and only fetches videos that aren't already saved, stopping pagination as soon as it hits a known video ID.
- **`main.py`** — orchestrates the full pipeline and writes the output to `frontend/public/categorized_videos.json`.

### 1. Get a YouTube Data API v3 key

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a project → Enable **YouTube Data API v3**
3. Create an **API Key** credential

### 2. Install dependencies and run

```bash
cd backend
pip install -r requirements.txt

# Set your API key
export YOUTUBE_API_KEY=YOUR_KEY_HERE

# Run the full pipeline
python main.py
```

This writes `frontend/public/categorized_videos.json`.

### Automating updates

Schedule `main.py` with a cron job or GitHub Actions to keep the data fresh:

```yaml
# .github/workflows/update-videos.yml
name: Update Videos
on:
  schedule:
    - cron: '0 6 * * *'   # every day at 06:00 UTC
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }
      - run: pip install google-api-python-client python-dotenv
      - run: python backend/main.py
        env:
          YOUTUBE_API_KEY: ${{ secrets.YOUTUBE_API_KEY }}
      - uses: stefanzweifel/git-auto-commit-action@v5
        with:
          commit_message: "chore: update categorized_videos.json"
          file_pattern: frontend/public/categorized_videos.json
```

## Frontend Setup

```bash
cd frontend
npm install
npm run dev          # local dev server at http://localhost:5173
npm run build        # production build → dist/
```

## Deploy to Vercel

### Option A — Vercel CLI

```bash
npm i -g vercel
cd frontend
vercel --prod
```

### Option B — GitHub integration

1. Push this repo to GitHub
2. Go to [vercel.com](https://vercel.com) → **Add New Project**
3. Import the repo; set **Root Directory** to `frontend`
4. Framework: **Vite**
5. Deploy!

Vercel automatically serves `frontend/public/categorized_videos.json` as a static asset.

---

## Updating data

After running `main.py`, commit and push `categorized_videos.json`. Vercel will redeploy automatically if you have GitHub integration enabled.
